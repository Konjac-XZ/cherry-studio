不要像操作普通文件那样把下载好的文件复制到SD卡中。你需要**将.img镜像写入/烧录到整张microSD卡**，这个操作会擦除卡内的所有数据。

## 最简单的方法：balenaEtcher

1. 将microSD卡插入你的电脑。
2. 安装/打开**balenaEtcher**。它支持Windows、macOS和Linux系统。（[Balena Etcher](https://etcher-docs.balena.io/)）
3. 点击**Flash from file**。
4. 选择你下载好的PYNQ镜像。
   - 如果镜像是.zip压缩包，若Etcher无法识别，请先解压。
   - 解压后你会得到类似`pynq_z1_*.img`、`pynq_z2_*.img`这类文件。
5. 点击**Select target**并选择你的**microSD卡**。
6. 点击**Flash**并等待验证完成。
7. 安全弹出存储卡。
8. 将卡插入Zynq/PYNQ开发板，如果开发板有启动跳针/启动开关，将其设置为**SD**启动模式，然后通电开机。

PYNQ官方文档也介绍了在Windows上使用Win32DiskImager、在macOS/Linux上使用`dd`的方法，但通常Etcher更安全，因为它可以帮你避免意外覆盖你的硬盘。PYNQ的文档特别提醒一定要仔细选择SD卡，因为选错驱动器会覆盖原有数据。([PYNQ](https://pynq.readthedocs.io/en/latest/appendix/sdcard.html))

## Linux命令行方式

请仔细替换镜像路径和设备名称：

```bash
lsblk
```

插入SD卡，然后再次运行该命令：

```bash
lsblk
```

找到整张磁盘，例如`/dev/sdb`或`/dev/mmcblk0`，**不要**选择类似`/dev/sdb1`这样的分区。

卸载所有已挂载的分区：

```bash
sudo umount /dev/sdb1
```

烧录镜像：

```bash
sudo dd bs=4M if=/path/to/pynq_image.img of=/dev/sdb status=progress conv=fsync
```

然后弹出设备：

```bash
sync
sudo eject /dev/sdb
```

PYNQ的Linux说明中明确要求写入整个microSD设备，而不是单个分区，并且采用的也是基本相同的`dd`方法。([PYNQ](https://pynq.readthedocs.io/en/latest/appendix/sdcard.html))

## macOS命令行方式

```bash
diskutil list
```

找到SD卡，例如`disk4`，然后执行：

```bash
diskutil unmountDisk /dev/disk4
sudo dd bs=1m if=/path/to/pynq_image.img of=/dev/rdisk4
sync
diskutil eject /dev/disk4
```

使用`disk4`，而不是`disk4s1`。PYNQ的macOS说明步骤为：先使用`diskutil list`，卸载磁盘，再通过`dd`写入`/dev/rdisk...`。([PYNQ](https://pynq.readthedocs.io/en/latest/appendix/sdcard.html))

烧录完成后，你的电脑可能会提示SD卡不可读，或者只显示一个很小的启动分区。这通常是正常现象，开发板仍然可以从该卡启动。
